HTTP - Introducción

El protocolo HTTP - Hypertext Transfer Protocol, es el reglamento de comunicación utilizado para el acceso y distribución de contenido en la World Wide Web. Este pertenece a la capa de aplicación (Modelo OSI) y fue diseñado para establecer reglas de comunicación entre servidores y clientes.

HTTP permite extraer recursos desde un servidor, tales como páginas html, archivos .json, .xml y otros formatos de extensión. Usualmente, las páginas están conformadas por hipertexto y multimedia que facilitan la interacción del usuario con el sitio web. La interactividad permite que se puedan solicitar datos al servidor y de esta manera realizar determinadas tareas.

Características

  • Simple: Los mensajes enviados a través del protocolo HTTP pueden ser leídos y entendidos fácilmente por los desarrolladores.
  • Extensible: Permite agregar funcionalidad mediante HTTP headers.
  • Stateless: No existe vínculo entre las peticiones realizadas; es decir, el estado actual del sitio se pierde al solicitar otro recurso. No obstante, el protocolo permite almacenar estos estados del lado del cliente mediante cookies.

Mensajes

Existen dos tipos de mensajes: Peticiones (Requests) y respuestas (Responses), cada una con su propio formato.

Peticiones (Requests)

Una petición está conformada por los siguientes elementos:

  • Método HTTP: Define la operación que el cliente desea realizar. Ejemplo: GET
  • Ruta: La ruta (Path) donde se encuentra alojado el recurso. Ejemplo: /about/
  • Versión: La versión del protocolo HTTP (1.1 ó 2.0), Ejemplo: HTTP/1.1
  • Cabeceras HTTP: Las Cabeceras (Headers) envían otra información al servidor y añaden funcionalidad. Ejemplo: accept
  • Cuerpo: Utilizado en algunos métodos, como en POST; contiene el recurso enviado al servidor.

Respuestas (Responses)

Una respuesta está conformada por los siguientes elementos:

  • Versión: La versión del protocolo HTTP. Ejemplo: HTTP/1.1
  • Código de estado (Status Code): Un código que indica si la petición fue satisfactoria o no, y el porqué. Ejemplo: 200
  • Mensaje del estado: Una breve descripción del estado. Ejemplo: OK
  • Cabeceras: Similar a las peticiones. Ejemplo: ‘Date’, ‘Content-Type’
  • Cuerpo: Opcionalmente, puede retornar un recurso solicitado. Ejemplo: HTML,JSON,XML,texto plano, etc…

Métodos HTTP

El protocolo HTTP define un conjunto de métodos, los cuales indican que acciones se van a realizar sobre un determinado recurso.

Método GET

Solicita la representación de un recurso; es decir, estas peticiones sólo deberían retornar datos.
Generalmente, se utilizan para solicitar el contenido de una página web, este recurso puede ser un archivo .html; el cual es procesado por el navegador para visualizarlo en la pantalla. También, pueden devolver datos en otros formatos como .json, .xml, o bien texto plano.

Objetivo: Recuperar un objeto

Método POST

Envía una entidad al recurso especificado causando cambios en el estado o efectos secundarios en el servidor. Usualmente, se utiliza para agregar nuevos registros en una colección, la cual puede estar almacenada en una base de datos. Se caracteriza por tener un cuerpo en el mensaje que contiene a la entidad.

Objetivo: Añadir un objeto

Método PUT

Reemplaza los atributos del recurso especificado con la entidad enviada (payload).

Objetivo: Actualizar un objeto

Método DELETE

Elimina el recurso especificado. Puede o no enviar datos en el cuerpo del mensaje.

Objetivo: Eliminar un objeto

Método PATCH

Aplica modificaciones parciales a un recurso. Mayormente, se utiliza para no enviar la entidad completa en el cuerpo del mensaje; es decir, para actualizar sólo algunos atributos.

Objetivo: Actualizar el atributo de un objeto

Método HEAD

Es similar al método GET, pero no retorna el cuerpo de la respuesta (En el mensaje); sólo las cabeceras.

Objetivo: Obtener datos de cabecera de una forma más rapida al método GET.
